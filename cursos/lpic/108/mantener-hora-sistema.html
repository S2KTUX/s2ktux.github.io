108.1 Mantener la hora del sistema

Configura la hora del sistema, zonas horarias, y sincronización con NTP y Chrony en Linux.

El comando date, como comentamos en la anterior unidad, muestra la hora y fecha actual del sistema. También permite ajustarla manualmente con el formato adecuado.

Reloj de hardware

El comando hwclock interactúa con el reloj en tiempo real (RTC) del hardware, que mantiene la hora incluso cuando el sistema está apagado. Este comando se utiliza para sincronizar el reloj del hardware con el reloj del sistema

timedatectl ofrece una alternativa para verificar y configurar la hora, mostrando el estado de sincronización con la red.

Ejemplos:

  • Listar zonas: timedatectl list-timezones
  • Ajustar hora: timedatectl set-time "2025-05-24 12:30:00"

Zona horaria

La zona horaria define el desfase respecto a UTC y ajustes como el horario de verano. Los datos de zonas horarias están en /usr/share/zoneinfo/.

Ejemplo de una configuración manual:

sudo ln -sf /usr/share/zoneinfo/Canada/Eastern /etc/localtime

El archivo /etc/timezone también puede especificar la zona horaria.

Una zona horaria correcta es esencial para convertir marcas de tiempo Unix a hora local, especialmente en regiones con horario de verano.

Demonio NTP

El protocolo NTP (Network Time Protocol) sincroniza la hora del sistema con servidores de red. El demonio ntpd se ejecuta en segundo plano para mantener la hora precisa.

El archivo de configuración es /etc/ntp.conf, donde se especifican servidores NTP (IP o URL, si DNS está configurado).

ntpd sondea múltiples servidores y selecciona los más precisos para ajustar la hora, corrigiendo desviaciones gradualmente.

Chrony

chrony es una alternativa moderna a ntpd, con mejor precisión y soporte para sincronización intermitente.

El archivo de configuración es /etc/chrony.conf. Hay que tener en cuenta que las opciones del archivo de configuración de chrony que definen las fuentes de tiempo remotas son pool y server

Ejemplo de configuración en /etc/chrony.conf:

pool pool.ntp.org iburst

La opción iburst acelera la sincronización inicial enviando múltiples paquetes al servidor NTP.
