107.3 Localización e internacionalización

Configura zonas horarias y localización en Linux para adaptarlo a diferentes regiones y idiomas.

La localización e internacionalización permiten adaptar Linux a diferentes idiomas, formatos y zonas horarias.

Zonas horarias

Zonas horarias son regiones con la misma hora, relativas a UTC (GMT+0). Se nombran por ciudades o desfases (Etc/GMT+3).

Configuración:

  • /etc/timezone: Define la zona horaria del sistema.
  • /etc/localtime: Enlace simbólico a un archivo en /usr/share/zoneinfo/ para ajustes como horario de verano. Ejemplo: sudo ln -sf /usr/share/zoneinfo/America/Sao_Paulo /etc/localtime
  • TZ: Variable de entorno para la zona horaria de la sesión. Ejemplo: export TZ="Africa/Cairo"; date

Comandos:

  • date: Muestra hora y zona. Tambien puede Ajustar la fecha y hora del sistema
  • timedatectl: Muestra/configura hora (systemd). Ejemplo: timedatectl set-timezone America/Sao_Paulo
  • tzselect: Selecciona zona horaria interactivamente. Ejemplo: tzselect → Seleccionar Brazil → Southeast → America/Sao_Paulo

Nota: Mantén /usr/share/zoneinfo/ actualizado para horarios de verano.

Localización

La localización configura idioma y formatos (moneda, números, etc.) mediante variables de entorno.

Algunas varibles a tener en cuenta son ::

  • LANG: Define idioma y codificación. Ejemplo: export LANG=es_ES.UTF-8
  • LC_MESSAGES: Idioma de mensajes. Ejemplo : export LC_MESSAGES="es"
  • LC_TIME: Formato de hora/fecha.
  • LC_ALL: Anula todas las variables.

Configuración:

  • /etc/locale.conf: Define localización del sistema.
  • locale: Muestra variables de localización.
  • localectl: Configura locale (systemd). Ejemplo: localectl set-locale LANG=es_US.UTF-8

Conversión de codificación

iconv: Es un comando que convierte codificaciones de caracteres, es decir, convierte archivos de un juego de caracteres a otro

Podemos listar codificaciones: iconv -l
