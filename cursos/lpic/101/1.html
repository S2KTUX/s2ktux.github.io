Introducción a la Arquitectura del Sistema

Este bloque del examen LPIC-1 se centra en la comprensión profunda de cómo Linux interactúa con el hardware físico y cómo se gestiona el ciclo de vida del sistema desde que se pulsa el botón de encendido hasta que el kernel está completamente operativo.

Aprenderás a inspeccionar dispositivos, gestionar módulos del kernel y entender las diferencias fundamentales entre sistemas de herencia (SysVinit) y modernos (Systemd).

101.1 Determinar y configurar los ajustes de hardware

Para administrar un sistema, primero debemos conocer qué hay dentro. La interacción comienza en el firmware (BIOS/UEFI) y continúa en el sistema operativo mediante el kernel y sus módulos.

Activación en Firmware (BIOS/UEFI)

Antes de cargar Linux, podemos configurar periféricos integrados, proteger contra errores o ajustar DMA/IRQ desde la configuración de la placa base (teclas F2, Supr, Esc al inicio).

Inspección de Dispositivos en Linux

Linux identifica el hardware mediante el kernel. Podemos interrogar al sistema usando comandos específicos o leyendo el sistema de archivos virtual /proc y /sys.

  • lspci: Muestra dispositivos PCI. Usa -v para detalles y -k para ver el driver del kernel asociado.
  • lsusb: Muestra dispositivos USB. -t muestra un árbol jerárquico.
  • lsmod: Lista los módulos del kernel cargados actualmente. Esencial para saber si un driver está activo.
  • modprobe: Gestión inteligente de módulos.
    • modprobe -r modulo: Descargar (eliminar) módulo.
    • modprobe modulo: Cargar módulo.

Sistemas de archivos clave (/proc, /sys, /dev)

/proc
Info de procesos y sistema
/sys
Info de hardware y kernel
/dev
Archivos de dispositivos

Archivos importantes en /proc incluyen cpuinfo, interrupts, ioports y dma. Los dispositivos extraíbles son gestionados dinámicamente por udev bajo /dev.

Nomenclatura de Almacenamiento

Los dispositivos de bloque tienen nombres predecibles según su controlador:

/dev/hdX   # Canales IDE (Antiguo)
/dev/sdX   # SCSI / SATA / USB (Estándar actual)
/dev/nvme0nXpY # Discos NVMe (n=controlador, X=disco, Y=partición)
/dev/fdX   # Disquetes

101.2 Arranque del Sistema

El proceso de arranque es una secuencia crítica donde el firmware entrega el control al bootloader, y este al kernel.

BIOS vs UEFI

BIOS (Legacy) UEFI (Moderno)
1. POST 1. POST
2. Lee MBR (Primeros 512 bytes) 2. Lee partición EFI (ESP) y NVRAM
3. Carga 1ª etapa GRUB -> Kernel 3. Ejecuta aplicación EFI (GRUB) -> Kernel

El Cargador de Arranque (GRUB 2)

GRUB permite pasar parámetros al kernel en el momento del arranque. Los más comunes se editan en /etc/default/grub y se aplican con update-grub o grub-mkconfig.

  • init=/bin/bash : Cambiar el init system (para recuperación).
  • system.unit=multi-user.target : Arrancar en un modo específico.
  • quiet / splash : Ocultar mensajes de texto del kernel.

Inicialización del Kernel

Una vez cargado, el kernel descomprime el initramfs (sistema de archivos raíz temporal en RAM) para cargar módulos necesarios para montar el disco real (/). Luego ejecuta el primer proceso (PID 1), que hoy en día suele ser systemd (aunque puede ser SysVinit).

Inspección del Arranque

Para diagnosticar fallos de arranque, revisamos los registros del kernel:

dmesg                 # Muestra el ring buffer del kernel
dmesg | grep -i usb  # Filtrar por dispositivos USB
journalctl -b -1     # Ver logs del arranque ANTERIOR (systemd)
journalctl -p err    # Ver solo errores

101.3 Cambiar runlevels / targets y apagar el sistema

Estándar SysVinit (Runlevels)

En sistemas heredados, el estado del sistema se define por un número (runlevel) configurado en /etc/inittab.

0
Apagado (Halt)
1
Modo Usuario Único (Recovery)
2-4
Multiusuario (sin gráficos usualmente)
5
Multiusuario Gráfico
6
Reinicio (Reboot)

Comandos clave: telinit 5 (cambiar nivel), runlevel (ver actual/anterior).

Systemd (Targets)

Systemd reemplaza los números numéricos por "unidades objetivo" (targets). Es más potente y gestiona dependencias.

# Equivalencias
poweroff.target   -> Runlevel 0
rescue.target     -> Runlevel 1
multi-user.target -> Runlevel 3
graphical.target  -> Runlevel 5
reboot.target     -> Runlevel 6

Gestión básica con systemctl:

systemctl isolate multi-user.target  # Cambiar modo ahora
systemctl set-default graphical.target # Establecer modo por defecto
systemctl get-default                 # Ver modo por defecto

Apagado y Reinicio Seguro

Nunca desconectues la energía directamente. Usa comandos que signalen a los procesos que terminen:

  • shutdown -h now : Apagar inmediatamente.
  • shutdown -r +10 : Reiniciar en 10 minutos.
  • systemctl poweroff : Método systemd.
  • systemctl reboot : Método systemd.
  • wall "Mantenimiento programado" : Enviar mensaje a todos los usuarios logueados.
