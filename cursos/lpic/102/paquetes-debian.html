Gestión con dpkg y apt

Para gestionar paquetes en Debian, suelo usar dos herramientas principales: dpkg, que es de bajo nivel y trabaja directamente con archivos .deb, y apt, que es de alto nivel, resuelve dependencias y accede a repositorios. Aquí tienes los comandos clave para cada una.

Comandos con dpkg

Acción Comando
Instalar un paquete dpkg -i paquete.deb
Eliminar un paquete dpkg -r paquete
Purgar un paquete (incluye configuración) dpkg --purge paquete
Listar paquetes instalados dpkg -l
Ver información de un paquete dpkg -s paquete
Listar archivos de un paquete dpkg -L paquete
Buscar a qué paquete pertenece un archivo dpkg -S /ruta/archivo
Configurar paquetes pendientes dpkg --configure -a

Comandos con apt

Acción Comando
Instalar un paquete apt install paquete
Actualizar el índice de paquetes apt update
Actualizar todos los paquetes apt upgrade
Eliminar un paquete apt remove paquete
Purgar un paquete apt purge paquete
Limpiar paquetes descargados apt autoclean
Eliminar dependencias no usadas apt autoremove
Buscar un paquete apt search paquete
Ver información de un paquete apt show paquete

Configuración de repositorios

En Debian, configuro los repositorios en /etc/apt/sources.list y en el directorio /etc/apt/sources.list.d/. Cada línea en estos archivos sigue un formato específico: deb [opciones] URL distribucion componentes. Por ejemplo:

  • deb http://deb.debian.org/debian bullseye main contrib non-free (para paquetes binarios).
  • deb-src http://deb.debian.org/debian bullseye main contrib non-free (para código fuente).

Después de editar, siempre actualizo el índice con apt update. Si quiero añadir un repositorio, puedo crear un archivo en /etc/apt/sources.list.d/, como mi-repo.list, y luego ejecutar apt update.
