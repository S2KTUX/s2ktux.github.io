102.6 Linux como sistema virtualizado

Juntos vamos a descubrir cómo funciona Linux en entornos virtualizados: qué es la virtualización y cómo usarla en la nube.

Descripción general de virtualización

Usamos la virtualización para crear máquinas virtuales que actúan como computadoras completas dentro de una sola máquina física. Esto lo hacemos con un programa especial llamado hipervisor, que se encarga de repartir los recursos del equipo (como la CPU o la RAM) entre las máquinas virtuales, también conocidas como "invitados". Así podemos tener varios sistemas operativos funcionando al mismo tiempo.

En Linux, los hipervisores más comunes que usamos son: Xen (Tipo 1, que funciona directamente en el equipo), KVM (Tipo 2, un hipervisor hospedado integrado en el núcleo de Linux que requiere un sistema operativo subyacente), y VirtualBox (Tipo 2, que se instala como una aplicación).

Tipos de Máquinas Virtuales

Se pueden crear tres tipos principales de máquinas virtuales:

  • Totalmente virtualizadas: Estas máquinas no saben que están en un entorno virtual. Funcionan como si fueran una computadora real, pero necesitan hardware especial (como Intel VT-x o AMD-V).
  • Paravirtualizadas: Estas sí saben que son virtuales y usan controladores especiales para trabajar mejor con el hipervisor, lo que mejora su rendimiento.
  • Híbridas: Combinamos lo mejor de las dos anteriores: usamos un sistema totalmente virtualizado, pero con controladores paravirtualizados para que el rendimiento sea casi como el de un equipo real.

Trabajando con plantillas de máquinas virtuales

Cuando necesitamos crear varias máquinas virtuales iguales, usamos plantillas. Una plantilla es como una máquina virtual ya configurada con lo básico (sistema operativo y ajustes iniciales). La copiamos, hacemos pequeños cambios (como el nombre o la dirección de red), y ya tenemos una nueva máquina lista rápidamente.

Algo importante que revisamos es el D-Bus Machine ID, un identificador único que cada máquina virtual debe tener. Si clonamos una máquina, debemos asegurarnos de que este ID sea diferente para evitar problemas con el hipervisor.

Implementación de máquinas virtuales en la nube

Por ultimo, podemos usar máquinas virtuales en la nube con servicios como AWS o Azure (conocidos como IaaS). Aquí creamos y gestionamos nuestras máquinas virtuales a través de herramientas web. Algunos puntos clave que consideramos son: las instancias de computación (cuánto tiempo de CPU usamos), el almacenamiento (dónde guardamos los datos y su costo), y las redes (cómo configuramos subredes o firewalls).

Para acceder a estas máquinas, usamos OpenSSH con claves públicas y privadas, lo que nos permite conectarnos de forma segura. También podemos usar cloud-init, una herramienta que nos ayuda a configurar automáticamente las máquinas al crearlas (por ejemplo, ajustando la red o instalando paquetes).
