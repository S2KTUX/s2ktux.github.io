106.2 Escritorios gráficos

Aprende sobre entornos de escritorio en Linux.

Los escritorios gráficos en Linux se basan en X11 y ofrecen interfaces visuales con entornos como Gnome, KDE y Xfce.

Sistema X Window

X11 es la base gráfica en Linux, manejando:

  • Eventos de entrada (ratón, teclado).
  • Copiar y pegar entre aplicaciones.
  • Interfaz para dibujar elementos gráficos.

No crea efectos visuales complejos; las aplicaciones lo hacen. Esto permite personalización, pero puede generar inconsistencias. La variable de entorno DISPLAY indica a las aplicaciones qué servidor X11 usar (por ejemplo, :0 para el local o localhost:10.0 para uno remoto), configurándose automáticamente en sesiones SSH con X11 forwarding. En sistemas con múltiples servidores X11, cada instancia se identifica con un nombre de pantalla como :0 o :1, usado por las aplicaciones para conectarse al servidor correcto. El archivo de configuración xorg.conf organiza sus ajustes en secciones, comenzando con Section "NombreSección" y terminando con EndSection, definiendo configuraciones como monitores o dispositivos de entrada.

Ambientes de escritorio

Un entorno de escritorio incluye:

  • Gestor de ventanas: Controla posición, barras y botones (minimizar, maximizar, cerrar).
  • Widgets: Elementos como botones o campos de texto (usando bibliotecas como GTK+ o Qt).
  • Aplicaciones básicas: Terminal, gestor de archivos, navegador, calendario.

Ofrecen coherencia y facilitan el desarrollo de interfaces.

Entornos populares

Los principales entornos son:

  • Gnome: Popular en Fedora, Ubuntu. Usa Gnome Shell (pantalla completa) o Gnome Classic.
  • KDE: Usa KDE Plasma (openSUSE, Kubuntu). Basado en Qt, con herramientas para unificar estilos.
  • Xfce: Ligero y modular, ideal para equipos con pocos recursos.

Otros: Cinnamon, MATE, LXDE (bajo consumo).

Acceso remoto

X11 permite sesiones remotas con:

  • X11: Base del sistema gráfico.
  • XDMCP: Protocolo para pantallas remotas, poco usado por ancho de banda y seguridad.
  • VNC: Usa RFB, puertos 5900+, sin cifrado nativo (requiere VPN/SSH).
  • Spice: Para sistemas virtualizados, integra dispositivos locales. Permite conectar dispositivos USB locales, como memorias o cámaras, a aplicaciones en máquinas virtuales remotas y proporciona acceso a interfaces gráficas, optimizando la experiencia en entornos virtualizados. Está diseñado para acceder a la salida de video de máquinas virtuales, ofreciendo una experiencia gráfica fluida.
  • RDP: Acceso a Windows, soportado en Linux con clientes abiertos.

Herramienta: Remmina (soporta XDMCP, VNC, RDP, Spice).
