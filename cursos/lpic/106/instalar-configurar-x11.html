106.1 Instalar y configurar X11 (LPIC-1 106)

Aprende cómo funciona y se configura el sistema X Window en Linux.

← Volver a Tema 106

Descripción

El subtema 106.1 de LPIC-1 te enseña a instalar y configurar X11, el sistema base para interfaces gráficas en Linux. Entenderás su arquitectura, cómo funciona en red, y cómo personalizarlo con archivos de configuración. Es clave para gestionar entornos gráficos de manera efectiva.

Conceptos que aprenderás

Exploraremos los fundamentos y la configuración de X11 de forma sencilla:

  • Arquitectura de sistema X Window: X11 permite dibujar formas 2D (y 3D con extensiones) en la pantalla. Se divide en un **cliente** (aplicaciones como navegadores o terminales) y un **servidor** (que muestra el contenido). Normalmente, ambos están en el mismo equipo, pero pueden funcionar en red.
  • Estructura básica de X Window: Es modular y soporta redes. Clientes de diferentes computadoras pueden usar un servidor remoto. Nuevas funciones se añaden como **extensiones** (ej. `libXrandr`, `libX11`) sin cambiar el núcleo de X11.
  • Administradores de pantalla: Inician el servidor X tras el arranque. Ejemplos incluyen **GDM**, **SDDM** y **LightDM**. Mantienen el servidor activo y gestionan sesiones.
  • Nombres de pantalla: Identifican sesiones X con el formato hostname:displaynumber.screennumber. Ejemplo: :0 (local, primera pantalla). Si no hay hostname, es el equipo local.
  • Variable de entorno DISPLAY: Almacena el nombre de la pantalla actual (ej. echo $DISPLAY → :0). Permite iniciar apps en pantallas específicas (ej. DISPLAY=:0.1 firefox &).
  • Configuración de un servidor X: Tradicionalmente usa el archivo **/etc/X11/xorg.conf**, pero en distribuciones modernas, X se configura solo al iniciarse.
  • Secciones de xorg.conf: Incluyen **InputDevice** (teclados, ratones), **InputClass** (clases de dispositivos), **Monitor** (configuración de monitores), **Device** (tarjetas gráficas), **Screen** (une Monitor y Device), y **ServerLayout** (agrupa todo).
  • Creación de xorg.conf: Usa sudo Xorg -configure para generar xorg.conf.new, que luego mueves a **/etc/X11/xorg.conf**.
  • Wayland: Nuevo protocolo que reemplaza X11, más ligero. Usa un **compositor** en lugar de un servidor y la variable **WAYLAND_DISPLAY** (ej. wayland-0).
  • Herramientas de diagnóstico: El comando xdpyinfo muestra detalles del servidor X (versión, extensiones, resolución).

Conceptos principales destacados

Estos son los elementos clave que debes dominar:

  • **/etc/X11/xorg.conf**: Archivo principal de configuración de X11 (si existe). Define secciones como InputDevice, Monitor, etc.
  • **/etc/X11/xorg.conf.d/**: Directorio para archivos de configuración adicionales, como 00-keyboard.conf para teclados.
  • ~/.xsession-errors: Archivo de log donde se guardan errores de sesiones X, útil para depurar problemas.
  • xhost: Comando para controlar el acceso a un servidor X (ej. xhost + permite conexiones desde cualquier host).
  • xauth: Gestiona la autenticación para conexiones X remotas, usando claves en archivos .Xauthority.
  • DISPLAY: Variable de entorno que indica la pantalla en uso (ej. :0 o :0.1).
  • X: Comando o enlace a Xorg para iniciar o configurar el servidor X.
