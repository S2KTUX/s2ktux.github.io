106.1 Instalar y configurar X11

Aprende cómo funciona y se configura el sistema X Window en Linux.

X11, o X Window System, es la base para interfaces gráficas en Linux. Aprender a configurarlo es esencial para gestionar entornos gráficos.

Arquitectura de sistema X Window

X11 es un sistema que permite mostrar gráficos en pantalla, como ventanas, formas 2D y 3D (con extensiones). Se divide en:

  • Cliente: Aplicaciones gráficas (ej. navegador web, terminal, juego). El cliente decide qué dibujar y dónde (ubicación y tamaño de la ventana).
  • Servidor: Gestiona la pantalla y dibuja lo que pide el cliente. También maneja dispositivos de entrada como teclado, ratón y trackpads.
  • Funcionamiento: En la mayoría de los casos, cliente y servidor están en el mismo equipo, pero X11 puede trabajar en red, permitiendo que un cliente remoto use un servidor en otro equipo.

Estructura básica de X Window

X11 es modular y puede operar en red. Aquí sus características principales:

  • Funcionamiento en red: Múltiples clientes X en diferentes computadoras pueden conectarse a un servidor X remoto. Esto permite usar aplicaciones gráficas en sistemas remotos.
  • Extensiones: Nuevas funciones se añaden como extensiones (ej. libXrandr, libXcursor, libX11, libxkbfile), manteniendo el núcleo del protocolo X11 intacto.
  • Administradores de pantalla: Inician el servidor X al arrancar. Ejemplos: GDM, SDDM, LightDM. Gestionan sesiones y mantienen el servidor activo.
  • Nombres de pantalla: Identifican sesiones X con el formato hostname:displaynumber.screennumber.
    • hostname: Nombre del equipo (si falta, es local).
    • displaynumber: Número de la sesión (empieza en 0).
    • screennumber: Número de pantalla (0 si es una sola pantalla lógica).
    • Ejemplo: :0 (local, primera pantalla).
  • Configuraciones de pantalla:
    • Un solo monitor: :0.
    • Dos monitores como una pantalla lógica: Ventanas se mueven libremente.
    • Dos monitores independientes: :0.0 y :0.1, ventanas no se mueven entre pantallas.

Configuración de un servidor X

Configurar X11 implica ajustar cómo interactúa con el hardware y las pantallas. Aquí los puntos clave:

  • Archivo principal: Tradicionalmente, X11 se configura con /etc/X11/xorg.conf, pero en sistemas modernos se autoconfigura al iniciar.
  • Generar configuración: Usa el comando: 
    sudo Xorg -configure
    Genera xorg.conf.new. Muévelo a /etc/X11/xorg.conf con: 
    sudo mv xorg.conf.new /etc/X11/xorg.conf
    Si hay una sesión activa, especifica un display diferente (ej. sudo Xorg :1 -configure).
  • Secciones de xorg.conf:
    • InputDevice: Configura dispositivos específicos (teclado, ratón).
    • InputClass: Configura clases de dispositivos (ej. todos los teclados). Ejemplo en /etc/X11/xorg.conf.d/00-keyboard.conf: 
      Section "InputClass"
    Identifier "system-keyboard"
    MatchIsKeyboard "on"
    Option "XkbLayout" "us"
    Option "XkbModel" "pc105"
EndSection
      Cambia el diseño con setxkbmap (temporal) o localectl (permanente): 
      setxkbmap -model chromebook -layout "gr(polytonic)"
localectl --no-convert set-x11-keymap "gr(polytonic)" chromebook
    • Monitor: Define el monitor físico. Ejemplo: 
      Section "Monitor"
    Identifier "DP2"
    Option "Primary" "true"
EndSection
    • Device: Configura la tarjeta gráfica. Ejemplo: 
      Section "Device"
    Identifier "Device0"
    Driver "i915"
    BusID "PCI:0:2:0"
EndSection
    • Screen: Une Monitor y Device. Ejemplo: 
      Section "Screen"
    Identifier "Screen0"
    Device "Device0"
    Monitor "DP2"
EndSection
    • ServerLayout: Agrupa todo (pantallas, teclado, ratón). Ejemplo: 
      Section "ServerLayout"
    Identifier "Layout-1"
    Screen "Screen0" 0 0
    InputDevice "mouse1" "CorePointer"
    InputDevice "system-keyboard" "CoreKeyboard"
EndSection
  • Otros archivos: Configuraciones adicionales están en /etc/X11/xorg.conf.d/ (prioridad) y /usr/share/X11/xorg.conf.d/.
  • Diagnóstico: Usa xdpyinfo para ver detalles del servidor X (versión, extensiones, resolución). Ejemplo: 
    xdpyinfo
name of display: :0
version number: 11.0
vendor string: The X.Org Foundation
...
number of screens: 1

Conceptos principales

Estos son los elementos clave que debes dominar para el examen:

  • /etc/X11/xorg.conf: Archivo principal de configuración de X11. Contiene secciones como InputDevice, Monitor, Device, Screen y ServerLayout.
  • /etc/X11/xorg.conf.d/: Directorio para configuraciones adicionales, como 00-keyboard.conf. Tiene prioridad sobre xorg.conf.
  • ~/.xsession-errors: Archivo de log donde se registran errores de la sesión X. Útil para diagnosticar problemas.
  • xhost: Controla el acceso al servidor X. Ejemplo: xhost + permite conexiones desde cualquier host (cuidado, puede ser inseguro).
  • xauth: Gestiona autenticación para conexiones remotas. Usa claves en ~/.Xauthority. Ejemplo: xauth list muestra las claves.
  • DISPLAY: Variable de entorno que indica la pantalla en uso. Ejemplo: echo $DISPLAY → :0. Para abrir una app en otra pantalla: DISPLAY=:0.1 firefox &.
  • X: Comando (o enlace a Xorg) para iniciar o configurar el servidor X. Ejemplo: X -configure.

Introducción a Wayland

Wayland es un protocolo moderno que reemplaza a X11, más ligero y eficiente:

  • Diferencia con X11: No usa un servidor entre cliente y kernel. El cliente renderiza directamente con un compositor Wayland, que gestiona ventanas y entrada.
  • Variable: Usa WAYLAND_DISPLAY (ej. echo $WAYLAND_DISPLAY → wayland-0).
  • Compatibilidad: Apps de X11 pueden ejecutarse en Wayland usando XWayland, un servidor X dentro de Wayland.
