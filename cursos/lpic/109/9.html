109.1 Fundamentos de los protocolos de Internet

TCP/IP es una pila de protocolos que permite la comunicación entre dispositivos mediante protocolos como IP, TCP, UDP, DNS y SMTP.

IPv4: Direccionamiento y máscaras

IPv4 es el protocolo de direccionamiento lógico que asigna direcciones únicas (32 bits) para enviar paquetes entre hosts.

  • Estructura: 32 bits divididos en 4 octetos (ej. 192.168.10.20). En binario: `11000000.10101000.00001010.00010100`.
  • Clases de direcciones:
    • Clase A: 1-126 (Redes grandes).
    • Clase B: 128-191 (Redes medianas).
    • Clase C: 192-223 (Redes pequeñas).
  • Máscara de red: Define qué parte es red y qué parte es host (ej. `255.255.0.0` /24).
  • Reservados (Privados):** No enrutables en internet.
    • 10.0.0.0/8: Redes grandes (10.0.0.0 a 10.255.255.255).
    • 172.16.0.0/12: Redes medianas (172.16.0.0 a 172.31.255.255).
    • 192.168.0.0/16: Redes pequeñas (192.168.0.0 a 192.168.255.255).

Protocolos: TCP, UDP e ICMP

Protocolos de transporte y red para la comunicación.

  • Puertos: 16 bits (1-65535).
    • 1-1023: Puertos privilegiados (ej. 80 HTTP, 53 DNS, 443 HTTPS).
    • 1024-65535: Puertos no privilegiados.
  • TCP: Orientado a conexión, garantiza entrega ordenada y retransmisión. Lento pero fiable (ideal para correo).
  • UDP: Sin conexión, rápido, sin control de errores (ideal para streaming).
  • ICMP: Analiza y controla la red. ping usa esto para verificar conectividad (Echo Request/Reply).

IPv6: Direcciones y diferencias

IPv6 usa direcciones de 128 bits.

  • Formato: 8 grupos de 16 bits en hexadecimal (ej. `2001:db8:3241::1`).
  • Tipos: Unicast (Una interfaz), Multicast (Grupo), Anycast (Grupo).
  • Diferencias:
    • Puertos: IPv6 usa corchetes `[]` (ej. `[2001:0db8::1]:443`).
    • No admite broadcast, usa multicast en su lugar.
    • Hop Limit: Reemplaza al TTL de IPv4.

109.2 Configuración de red persistente

Las configuraciones de red se guardan en archivos en `/etc` para aplicar al arrancar.

Interfaces de red

Las interfaces (ej. `enp3s5` para Ethernet o `wlo1` para Wi-Fi) conectan el sistema a la red. `ip link show` las lista.

Los nombres modernos (Predictable Network Interface Names) se basan en su ubicación física (bus, slot).

Gestionar interfaces

  • ifup / ifdown: Activa/desactiva interfaces según `/etc/network/interfaces`.
    • Ejemplo estático: `iface enp3s5 inet static`, `address 192.168.1.2/24`.
  • Comandos:
    • Ver: ip link set dev enp0s8 up.
    • Asignar IP: ip addr add 192.168.5.5/24 dev enp0s8.

Nombres y resolución

  • Nombre host: `/etc/hostname` define el nombre estático. Cambio permanente: hostnamectl set-hostname nuevo-nombre.
  • Mapeo local: `/etc/hosts` (ej. `127.0.0.1 localhost`).
  • Resolución: `/etc/nsswitch.conf` define el orden (ej. `hosts: files dns`).
  • DNS: `/etc/resolv.conf` define `nameserver 8.8.8.8` y `search lpi.org`.

NetworkManager y systemd

  • nmcli: Gestiona conexiones (ej. `nmcli connection show`).
  • nmtui: Interfaz curses para ajustes interactivos.
  • systemd-networkd: Usa archivos `.network` en `/etc/systemd/network` (ej. Address=192.168.0.100/24).

109.3 Resolución de problemas básicos de red

El diagnóstico se realiza con estas herramientas clave:

Configurar tu red

  • Ver interfaces: ip link show (muestra `enp0s3`, `lo`).
  • Activar: ip link set dev enp0s8 up.
  • Asignar IP: ip addr add 192.168.5.5/24 dev enp0s8.

Herramientas para buscar problemas

  • ping / ping6: Prueba conectividad ICMP. ping 8.8.8.8.
  • traceroute / traceroute6: Rastrea la ruta mostrando saltos (routers).
  • netcat: Prueba puertos. nc -zv 192.168.1.1 80 (si muestra "open", el puerto está abierto).

Ver y entender las rutas

  • Tabla de enrutamiento: ip route.
  • Ejemplo de salida: `default via 10.0.2.2 dev enp0s3 proto dhcp metric 100`.
  • Añadir ruta: ip route add 192.168.10.0/24 via 192.168.1.1.
  • IPv6: ip -6 route` (muestra prefijos de red como `2001:db8::/64`).

Ver conexiones con netstat y ss

Monitorea conexiones activas y puertos.

  • netstat -t: Conexiones TCP (ej. `ESTAB 0 0 192.168.1.100:22`).
  • netstat -l: Puertos en escucha.
  • netstat -p: Muestra el proceso (ej. `1234/sshd`).
  • ss: Alternativa moderna. Usa `ss -tulnp` para ver todo: TCP/UDP, escuchando, numéricos y programas (ej. `nginx` en puerto 80).

109.4 Configuración DNS en el lado del cliente

La resolución de nombres convierte "google.com" en una IP.

Cómo se buscan los nombres

El proceso sigue el orden definido en `/etc/nsswitch.conf` (ej. `hosts: files dns`).

Archivos importantes

  • /etc/hosts: Mapeo estático. Ej: `127.0.0.1 localhost` o `192.168.1.10 foo.mydomain.org`.
  • /etc/resolv.conf: Servidores DNS (`nameserver 8.8.8.8`) y dominios de búsqueda (`search lpi.org`).
  • /etc/nsswitch.conf: Define el orden (si buscar primero en archivos locales o en DNS).

Herramientas útiles

  • host: Consultas simples. Ej: `host google.com` (muestra IP).
  • dig: Consultas detalladas. Ej: `dig google.com` (muestra IP, TTL, servidores).
  • getent: Busca según /etc/nsswitch.conf`. Ej: `getent hosts google.com` (muestra IP encontrada).
