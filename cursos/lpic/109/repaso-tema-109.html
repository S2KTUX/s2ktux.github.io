109.1 - Fundamentos de protocolos de internet

  • TCP (confiable, 3-way handshake) y UDP (rápido).
  • Capas: Red (IP), Transporte (TCP/UDP), Aplicación (HTTP, FTP).
  • IPs: IPv4 (ej. 192.168.1.1), IPv6 (ej. 2001:db8::1), máscara /24.
  • Puertos: 80 (HTTP), 443 (HTTPS), 22 (SSH), 21 (FTP).
  • Subredes: ej. 203.0.113.64/28, hosts 203.0.113.65-78.

109.2 - Configuración de red persistente

  • /etc/hostname: Nombre (cambia con hostnamectl set-hostname).
  • /etc/hosts: IPs a nombres (ej. 127.0.0.1 localhost).
  • /etc/nsswitch.conf: Orden (ej. hosts: files dns).
  • /etc/resolv.conf: DNS (ej. nameserver 8.8.8.8).
  • Interfaces: /etc/network/interfaces, DHCP (iface enp3s5 inet dhcp), estático (address 192.168.1.2/24).
  • ifup enp3s5 activa, ifdown enp3s5 desactiva.
  • nmcli: connection (ej. show), device (ej. wifi connect).
  • nmtui: Ajustes interactivos.

109.3 - Resolución de problemas básicos de red

  • ping: ICMP Echo Request/Response (ej. ping 8.8.8.8).
  • traceroute: Rutas (ej. traceroute google.com).
  • netstat: Conexiones (-t), rutas (-r), estadísticas (-i).
  • ss: TCP (-t), UDP (-u).
  • Bytes en eth0: ip -s link show eth0 o ifconfig eth0.
  • ip addr: Configura (ej. ip addr add 192.168.1.10/24).
  • nmtui: Ajusta NetworkManager.
  • ip route: Verifica rutas.

109.4 - Conceptos de seguridad de red

  • Puertos: Cierra los innecesarios (443 HTTPS, 22 SSH).
  • Firewall: iptables o ufw allow ssh.
  • SSH: ssh user@host, usa claves.
  • nmap: Escanea puertos (ej. nmap 192.168.1.1).
  • Seguridad: Cifrado (SSL/TLS), autenticación, evita accesos no autorizados.
