109.2 Configuración de red persistente

Aprende a configurar redes en Linux para que funcionen al arrancar el sistema.

Introducción

En Linux, las configuraciones de red se guardan en archivos en `/etc` para que se apliquen al arrancar. Esto incluye interfaces, nombres de host y resolución de nombres, asegurando comunicación continua.

Interfaces de red

Las interfaces de red (como `enp3s5` para Ethernet o `wlo1` para Wi-Fi) conectan el sistema a la red. La interfaz `lo` (loopback) sirve para conexiones internas. Usa `ip link show` para listarlas, mostrando nombres como `enp3s5`.

Los nombres modernos usan prefijos: `en` (Ethernet), `wl` (Wi-Fi), basados en su ubicación en el hardware, como `enp3s5` (bus 3, ranura 5).

Gestionar interfaces

Configura interfaces para que sean persistentes al arrancar.

  • ifup e ifdown: Activan o desactivan interfaces según `/etc/network/interfaces`. Por ejemplo, con `auto enp3s5` e `iface enp3s5 inet dhcp`, usa `ifup enp3s5` para activarla y `ifdown enp3s5` para desactivarla.
  • Método estático: En `/etc/network/interfaces`, usa `iface enp3s5 inet static`, `address 192.168.1.2/24`, `gateway 192.168.1.1` para una IP fija.

Nombres y resolución

Configura el nombre del sistema y la resolución de nombres.

  • Nombre del host: El archivo `/etc/hostname` define el nombre estático. Cámbialo con `hostnamectl set-hostname storage`, que actualiza `/etc/hostname`.
  • Mapeo local: Usa `/etc/hosts` para asociar IPs con nombres, como `127.0.0.1 localhost` o `192.168.1.10 foo.mydomain.org`.
  • Resolución de nombres: `/etc/nsswitch.conf` define el orden de búsqueda, como `hosts: files dns` (primero archivos, luego DNS). `/etc/resolv.conf` configura servidores DNS, como `nameserver 8.8.8.8`.

NetworkManager y systemd

NetworkManager automatiza la configuración de red.

  • nmcli: Gestiona conexiones desde la línea de comandos. Algunos subcomandos son nmcli connection (para mostrar o gestionar conexiones) y nmcli device (para controlar dispositivos).
  • nmtui: Ofrece una interfaz curses para gestionar conexiones de NetworkManager de forma interactiva, útil para ajustes manuales.
  • systemd-networkd: Usa archivos `.network` en `/etc/systemd/network` para configuraciones avanzadas, como `Address=192.168.0.100/24` o `DHCP=yes`.
