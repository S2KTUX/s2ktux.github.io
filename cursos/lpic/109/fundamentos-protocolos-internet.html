109.1 Fundamentos de los protocolos de Internet

Descubre los conceptos clave de IPv4, IPv6, TCP, UDP e ICMP en redes.

TCP/IP es una pila de protocolos que permite la comunicación entre dispositivos mediante protocolos como IP, TCP, UDP, DNS y SMTP.

IPv4: Direccionamiento y máscaras

IPv4 es el protocolo de direccionamiento lógico que asigna direcciones únicas para enviar paquetes entre hosts.

  • Dirección IPv4: Se componen de 32 bits, divididos en 4 octetos (ej. 192.168.10.20, que en binario sería 11000000.10101000.00001010.00010100).
  • Clases de direcciones:
    • Clase A: 1-126 (redes grandes).
    • Clase B: 128-191 (redes medianas).
    • Clase C: 192-223 (redes pequeñas).
  • Máscara de red: Define la parte de red y hosts (ej. 255.255.0.0: 16 bits para red, 16 para hosts), permitiendo hasta 62 hosts utilizables en subredes como 192.168.2.128/26.
  • Direcciones reservadas:
    • Dirección de red: Identifica la red (primera del rango, ej. 192.168.1.0).
    • Dirección de broadcast: Envía a todos los hosts (última del rango, ej. 192.168.1.255).
  • Rangos privados: Reservados por IANA para redes internas, no enrutables en Internet. Incluyen 192.168.0.0/16 (192.168.0.0 a 192.168.255.255) para redes pequeñas.
    • 10.0.0.0/8: 10.0.0.0 a 10.255.255.255 (redes grandes).
    • 172.16.0.0/12: 172.16.0.0 a 172.31.255.255 (redes medianas).
    • 192.168.0.0/16: 192.168.0.0 a 192.168.255.255 (redes pequeñas).

Protocolos: TCP, UDP e ICMP

Los protocolos de transporte (TCP, UDP) y de red (ICMP) permiten la comunicación entre hosts mediante puertos.

  • Puertos: 16 bits, de 1 a 65535.
    • 1-1023: Puertos privilegiados (servicios, requieren root), como 443 para HTTPS, 80 para HTTP y 53 para DNS.
    • 1024-65535: Puertos no privilegiados (clientes).
  • TCP: Orientado a conexión, garantiza entrega ordenada y retransmite paquetes perdidos, ideal para correo. Diferencia clave: es más lento que UDP por su fiabilidad.
  • UDP: Sin conexión, más rápido, pero sin control de errores, ideal para streaming. Diferencia clave: no establece conexión previa, sacrificando fiabilidad por velocidad.
  • ICMP: Analiza y controla la red, usado por ping, que envía Echo Request y espera Echo Reply para verificar conectividad.

IPv6: Direcciones y diferencias

IPv6 usa direcciones de 128 bits y mejora las limitaciones de IPv4.

  • Dirección IPv6: 128 bits, 8 grupos de 16 bits en hexadecimal (ej. 2001:db8:3241::1 es válida), reduciendo 0000 a 0.
  • Tipos de direcciones:
    • Unicast: Identifica una interfaz (64 bits red, 64 bits interfaz).
    • Multicast: Envía a un grupo de interfaces (ej. ff02::1 para todos los hosts locales).
    • Anycast: Envía a una interfaz de un grupo.
  • Diferencias con IPv4:
    • Puertos: IPv6 usa corchetes (ej. [2001:0db8::1]:443).
    • IPv6 no admite broadcast, usa multicast como alternativa.
    • Hop Limit: Reemplaza el TTL de IPv4. Indica el número máximo de enrutadores que un paquete puede atravesar; decrece en 1 por cada enrutador que lo reenvía, evitando bucles infinitos.
