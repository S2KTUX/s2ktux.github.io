109.4 Configuración DNS en el lado del cliente

Aprende cómo hacer que tu computadora encuentre direcciones de internet fácilmente.

Introducción

Cuando usas internet, escribes nombres como "google.com" en lugar de números complicados como 142.250.190.14. La resolución de nombres es el proceso que convierte esos nombres en números para que tu equipo pueda conectarse.

Cómo se buscan los nombres

Tu equipo sigue un orden para buscar nombres. Primero mira un archivo que le dice cómo hacerlo, que se llama `/etc/nsswitch.conf`. Este archivo decide si busca en un archivo local o en un servidor DNS en internet.

  • Tipos de DNS: DNS tiene diferentes tipos o "clases". La más común es IN, que usamos para internet normal con direcciones como las de las páginas web. Hay otros tipos, como CH y HS, son muy viejos o poco usados.
  • Orden de búsqueda: En `/etc/nsswitch.conf`, una línea como `hosts: files dns` significa que primero busca en un archivo local y, si no lo encuentra, pregunta a un servidor DNS. Puedes cambiar este orden si quieres, por ejemplo, a `hosts: dns files`.

Archivos importantes

Hay tres archivos principales que tu equipo usa para encontrar nombres de páginas web.

  • /etc/hosts: Aquí mapeas nombres y sus direcciones IP directamente, por ejemplo, `127.0.0.1 localhost`. Sirve para cosas locales, como pruebas, y también puede incluir direcciones IPv6, como `::1 localhost`.
  • /etc/resolv.conf: Este archivo le dice a tu equipo a qué servidores DNS preguntar si no encuentra un nombre localmente. Por ejemplo, `nameserver 8.8.8.8` usa el servidor de Google, y `search lpi.org` añade ese dominio a nombres cortos, como convertir "aprendizaje" en "aprendizaje.lpi.org". Puedes poner hasta tres servidores.
  • /etc/nsswitch.conf: Define el orden y métodos de resolución. Por ejemplo, una línea como `hosts: files dns` le dice que mire primero en `/etc/hosts` y luego en DNS. También puede tener reglas especiales para manejar errores.

Herramientas útiles

Estas herramientas te ayudan a buscar nombres o solucionar problemas cuando no puedes conectarte a una página.

  • host: Es una herramienta sencilla para preguntar por un nombre. Por ejemplo, si escribes `host google.com`, te dice la dirección IP, como `142.250.190.14`, y otros datos, como los servidores de correo.
  • dig: Es como `host`, pero te da mucha más información, perfecta para encontrar problemas. Si escribes `dig google.com`, te muestra la dirección IP, cuánto tiempo es válida esa información y los servidores DNS que usó.
  • getent: Esta herramienta busca nombres siguiendo las reglas de `/etc/nsswitch.conf`. Por ejemplo, `getent hosts google.com` te muestra la dirección IP que tu computadora encuentra, ya sea en `/etc/hosts` o preguntando a un servidor DNS. Puedes decirle dónde buscar con `-s`, como `getent -s files hosts google.com` para mirar solo en `/etc/hosts`.
