103.1 Trabajar desde la línea de comandos

Obteniendo información del sistema

Estos comandos te ayudan a conocer detalles del sistema y del entorno de trabajo.

  • pwd: Muestra el directorio actual en el que estás.
    Ejemplo: Si estás en /home/usuario, al ejecutar pwd obtendrás: /home/usuario.
  • touch: Crea un archivo vacío.
    Ejemplo: touch archivo.txt crea un archivo llamado archivo.txt.
  • ls: Lista los archivos y carpetas del directorio actual.
    Ejemplo: ls podría mostrar: archivo.txt documentos imagen.jpg.
  • uname -a: Muestra información completa del sistema.
    Ejemplo: uname -a podría mostrar: Linux mi-pc 5.15.0-73-generic #80-Ubuntu SMP Mon May 15 15:11:47 UTC 2023 x86_64 x86_64 x86_64 GNU/Linux.

Obteniendo información de los comandos

Estos comandos te ayudan a aprender más sobre otros comandos y su uso.

  • man: Muestra el manual de un comando.
    Ejemplo: man ls muestra la documentación del comando ls.
  • apropos: Busca páginas del manual relacionadas con una palabra clave.
    Ejemplo: apropos copy podría mostrar comandos como cp (copiar archivos).
  • type: Indica qué tipo de comando es (integrado, alias, etc.).
    Ejemplo: type cd muestra: cd is a shell builtin.
  • hash -d: Limpia la caché de rutas de comandos.
    Ejemplo: Si cambias la ubicación de un comando, ejecuta hash -d ls para actualizar la caché de ls.
  • which: Muestra la ruta absoluta de un comando.
    Ejemplo: which ls podría mostrar: /bin/ls.
  • history: Lista los comandos que has ejecutado. También se guarda en el archivo .bash_history.
    Ejemplo: history podría mostrar: 1 ls 2 cd /home 3 pwd.

Encontrar, crear y eliminar variables de entorno

Las variables de entorno almacenan información útil para el sistema y los programas.

  • env: Muestra todas las variables de entorno.
    Ejemplo: env podría mostrar: PATH=/usr/bin:/bin HOME=/home/usuario.
  • echo $VARIABLE: Muestra el valor de una variable.
    Ejemplo: echo $HOME muestra: /home/usuario.
  • export VARIABLE: Crea o hace permanente una variable.
    Ejemplo: export MI_VAR="hola" crea la variable MI_VAR con el valor hola.
  • set: Lista todas las variables y funciones del entorno.
    Ejemplo: set podría mostrar: MI_VAR='hola' PATH='/usr/bin:/bin'.
  • unset: Elimina una variable de entorno.
    Ejemplo: unset MI_VAR elimina la variable MI_VAR.

Mantener el valor de caracteres especiales

En Bash, los caracteres alfanuméricos (a-z, 0-9) se leen literalmente, pero los caracteres especiales (como espacios o símbolos) necesitan un tratamiento especial.

Por ejemplo, si intentas crear un archivo con touch my big file, Bash interpretará los espacios como separadores. Para solucionarlo:

  • Comillas simples (''): Interpreta todo de forma literal.
    Ejemplo: touch 'my big file' crea my big file.
  • Comillas dobles (""): Interpreta algunos símbolos como $, `, \.
    Ejemplo: touch "my big file" crea my big file.
  • Contrabarra (\): Hace que Bash lea el siguiente carácter literalmente.
    Ejemplo: touch my\ big\ file crea my big file.

103.2 Procesar secuencias de texto usando filtros

Redirecciones y tuberías (pipes)

Los comandos de texto usan stdin (entrada), stdout (salida) y stderr (errores).

  • >: Redirige la salida a un archivo (sobrescribe).
    Ejemplo: echo "Hola" > saludo.txt.
  • >>: Añade la salida a un archivo.
    Ejemplo: echo "Adiós" >> saludo.txt.
  • | (pipe): Envía la salida de un comando a otro.
    Ejemplo: ls | grep ".txt".
  • diff: Compara dos archivos.
    Ejemplo: diff archivo1.txt archivo2.txt.
  • grep: Filtra texto. Con -i ignora mayúsculas/minúsculas.
    Ejemplo: grep -i "hola" saludo.txt.

Leer un archivo comprimido

  • bzip2 / bunzip2: Comprime y descomprime.
    Ejemplo: bzip2 archivo.txt crea archivo.txt.bz2.
  • gzip / gunzip: Comprime y descomprime.
    Ejemplo: gzip archivo.txt crea archivo.txt.gz.
  • xz / unxz: Comprime y descomprime.
    Ejemplo: xz archivo.txt crea archivo.txt.xz.
  • bzcat, zcat, xzcat: Muestra el contenido sin descomprimir.
    Ejemplo: zcat archivo.txt.gz.

Ver un archivo paginado

  • less: Muestra el archivo con navegación (arriba/abajo).
    Ejemplo: less archivo.txt.
  • more: Muestra el archivo página por página (solo hacia adelante).
    Ejemplo: more archivo.txt.

Obtener una parte del texto

  • head: Muestra las primeras líneas. Usa -n para elegir cuántas.
    Ejemplo: head -n 3 archivo.txt.
  • tail: Muestra las últimas líneas. Usa -n para elegir cuántas.
    Ejemplo: tail -n 2 archivo.txt.

Fundamentos de sed

sed filtra y transforma texto.

  • borrar: sed '/cat/d' nombres.txt elimina líneas con "cat".
  • imprimir coincidencias: sed -n '/cat/p' ftu.txt muestra solo líneas con "cat".
  • reemplazar: sed 's/cat/dog/' nombres.txt cambia "cat" por "dog".
  • reemplazar todas: sed -i 's/cat/dog/g' nombres.txt.
  • con backup: sed -i.backup 's/cat/dog/' nombres.txt.

Otros comandos útiles

  • nl: Numera las líneas del archivo.
    Ejemplo: nl archivo.txt.
  • wc: Cuenta líneas, palabras o caracteres. -l solo líneas.
    Ejemplo: wc -l archivo.txt.
  • cut: Extrae partes de cada línea.
    Ejemplo: cut -d' ' -f1 nombres.txt.
  • sort: Ordena líneas. -n ordena numéricamente.
    Ejemplo: sort -n numeros.txt.
  • tr: Sustituye caracteres.
    Ejemplo: echo "hola" | tr 'a' 'A' muestra holA.

103.3 Administración básica de archivos

Manipulación de archivos

En Linux, todo es un archivo: archivos regulares, directorios y archivos especiales.

  • ls -l: Muestra permisos, propietario, tamaño, fecha y nombre.
    Primer carácter: - (archivo), d (directorio), l (enlace simbólico).
  • Opciones útiles:
    • -a: Muestra archivos ocultos.
    • -h: Tamaños legibles (KB, MB).
    • -R: Lista directorios recursivamente.
  • touch: Crea un archivo vacío.
  • cp: Copia archivos. Usa -r para directorios.
  • mv: Mueve o renombra archivos.
  • rm: Elimina archivos. Usa -r para directorios.
  • mkdir: Crea directorios. Usa -p para crear subdirectorios.
  • rmdir: Elimina directorios vacíos.

Archivos globbing y wildcards

Comodines para coincidir con varios archivos.

  • *: Coincide con cero o más caracteres.
    Ejemplo: ls *.txt.
  • ?: Coincide con un solo carácter.
    Ejemplo: ls file?.txt.
  • []: Coincide con un carácter dentro de un rango.
    Ejemplo: ls file[1-3].txt.

Cómo encontrar archivos

Usa find para buscar archivos.

  • Por nombre: find / -name "archivo.txt" (o -iname para ignorar mayúsculas).
  • Por tipo: find / -type d (busca directorios).
  • Por tamaño: find /home -size +5M (mayores a 5 MB).
  • Por tiempo: find /tmp -mtime +2 (modificados hace más de 2 días).
  • Ejecutar acción: find /var/log -type f -name "*.log" -exec ls -l {} \;.

Archivado de archivos

Usa tar para agrupar archivos.

  • Crear: tar -cvf archivo.tar archivo1.txt archivo2.txt.
  • Extraer: tar -xvf archivo.tar.
  • Listar: tar -tvf archivo.tar.
  • Comprimir con gzip: tar -czvf comprimido.tar.gz dir1.
  • Comprimir con bzip2: tar -cjvf comprimido.tar.bz2 dir1.

103.4 Uso de secuencias de texto, tuberías y redireccionamientos

Redirecciones

  • <: Usa un archivo como entrada.
    Ejemplo: wc -l < archivo.txt.
  • 2>: Guarda errores en un archivo.
    Ejemplo: ls archivo_inexistente 2> errores.txt.
  • 2>>: Añade errores a un archivo.
  • &> o 2>&1: Guarda salida y errores juntos.
    Ejemplo: ls archivo_inexistente &> salida.txt.

Here Document y Here String

  • Here Document: Redirige varias líneas hasta un delimitador.
    Ejemplo: wc -c << EOF ... EOF.
  • Here String: Pasa una sola línea como entrada.
    Ejemplo: wc -c <<< "texto".

Tuberías (Pipes)

  • Ejemplo 1: cat /proc/cpuinfo | grep "model name" | uniq | cut -d ":" -f 2.
  • Ejemplo 2: ls *.txt | wc -l (cuenta archivos .txt).
  • Ejemplo 3: ps aux | grep "python" | sort -k 3 -r | head -n 3.

Sustitución de comando

  • Comillas invertidas ` `: archivos=`find . -type f | wc -l`.
  • $( ): echo "El proceso es $(ps ...)".
  • xargs: Pasa la salida como argumento.
    Ejemplo: find . -name "*.log" | xargs rm.

103.5 Crear, supervisar y matar procesos

Control de trabajos

  • jobs: Muestra trabajos corriendo o detenidos. -l incluye el PID.
  • fg: Trae un trabajo a primer plano.
    Ejemplo: fg %1.
  • bg: Envía un trabajo a segundo plano.
    Ejemplo: bg %1.
  • &: Ejecuta un comando en segundo plano.
  • nohup: Ejecuta un comando que sigue corriendo aunque cierres la sesión.
  • kill: Termina un proceso. -9 fuerza la terminación.

Monitoreo de procesos

  • watch: Ejecuta un comando cada X segundos.
    Ejemplo: watch -n 10 free -m.
  • pgrep: Busca procesos por nombre. pkill para terminarlos.
  • top: Muestra procesos en tiempo real. Ordena por CPU (defecto), memoria (M), tiempo (T).
  • ps: Muestra procesos de forma estática.
    Ejemplo: ps -aux o ps -ef.

Características de los multiplexores terminales

Herramientas como screen y tmux para trabajar con varias ventanas.

  • Screen: Ctrl + a c (crear ventana), Ctrl + a n (siguiente), Ctrl + a d (desconectar).
  • tmux: Ctrl + b c (crear ventana), Ctrl + b n (siguiente), Ctrl + b d (desconectar).

103.6 Modificar la prioridad de ejecución de los procesos

El planificador de Linux

Cada proceso tiene dos atributos:

  • Prioridad de ejecución (Nice): Valor de -20 (máxima) a 19 (mínima). Por defecto 0.
  • Política de programación: Tiempo real (mayor prioridad) o normales (menor prioridad).

Prioridades de ejecución

  • Tiempo real: 0 a 99.
  • Normales: 100 a 139 (120 es nice 0).

Para verificar prioridades: /proc/[pid]/sched, ps -el, top.

Niceness de procesos

  • nice: Inicia un proceso con un valor nice específico.
    Ejemplo: nice -n 10 sleep 100.
  • renice: Cambia el nice de un proceso en ejecución.
    Ejemplo: renice 15 1234 (solo root puede usar valores negativos).

103.7 Realizar búsquedas en archivos de texto usando expresiones regulares

Expresiones regulares básicas

  • .: Coincide con cualquier carácter.
  • ^: Inicio de línea.
  • $: Final de línea.
  • \: Interpreta carácter especial literal.

Expresiones de corchetes

  • [ab]: Coincide con "a" o "b".
  • [a-z]: Cualquier letra minúscula.
  • Clases: [:alpha:], [:digit:], [:lower:], [:upper:], [:blank:].

Cuantificadores y límites

  • *: 0 o más veces.
  • +: 1 o más veces (usa egrep o grep -E).
  • ?: 0 o 1 vez (usa egrep).
  • {N}: Exactamente N veces.
  • {N,}: Al menos N veces.
  • {N,M}: Entre N y M veces.

Búsqueda con grep

  • -c: Cuenta las coincidencias.
  • -i: Ignora mayúsculas/minúsculas.
  • -n: Muestra el número de línea.
  • -v: Muestra líneas que NO coinciden.
  • -r: Busca recursivamente.
  • egrep o grep -E: Expresiones extendidas.

103.8 Edición básica de archivos

Modos y comandos básicos de vi

vi tiene dos modos principales:

  • Modo normal: Navegar y ejecutar comandos.
  • Modo inserción: Escribir texto (presiona Esc para volver).

Comandos en modo normal:

  • h, j, k, l: Moverse.
  • 0, $: Inicio/Final de línea.
  • 1G, G: Inicio/Final del documento.
  • /texto: Buscar "texto".
  • i, a, o: Entrar en modo inserción.
  • x: Eliminar carácter.
  • yy: Copiar línea. p: Pegar.
  • u: Deshacer.

Comandos colon (:)

  • :w: Guardar.
  • :q: Salir.
  • :wq: Guardar y salir.
  • :q!: Salir sin guardar.
  • :s/texto/nuevo/g: Reemplazar en la línea actual.

Editores alternativos

  • nano: Ctrl+O (guardar), Ctrl+X (salir).
  • emacs: Ctrl+X Ctrl+S (guardar), Ctrl+X Ctrl+C (salir).
