104.6 Crear y cambiar enlaces duros y simbólicos

Aprende a crear y gestionar enlaces duros y simbólicos en Linux.

Tipos de enlaces

En Linux, un enlace es un tipo especial de archivo que apunta a otro archivo o directorio. Hay dos tipos principales:

  • Enlaces duros:
    - Apuntan al mismo inodo (identificador único de los datos en el disco).
    - Si se borra el archivo original, el enlace duro sigue funcionando porque los datos permanecen.
    - Solo se pueden crear dentro del mismo sistema de archivos.
  • Enlaces simbólicos (o blandos):
    - Son como un "atajo" que apunta a la ruta de otro archivo o directorio.
    - Si se borra el archivo original, el enlace se rompe.
    - Pueden crearse entre diferentes sistemas de archivos.

Gestión de enlaces

Los enlaces se crean y gestionan con el comando ln.

  • Enlaces duros:
    - Crear: ln archivo enlace.
    - Verificar: Usa ls -li para ver el inodo (el primer número debe ser el mismo para los enlaces duros).
    Ejemplo:
    $ ls -li
12345 -rw-r--r-- 2 user user 1024 Apr 27 12:00 archivo.txt
12345 -rw-r--r-- 2 user user 1024 Apr 27 12:00 enlace_duro.txt
    Ambos tienen el mismo inodo (12345) y el contador de enlaces es 2.
  • Enlaces simbólicos:
    - Crear: ln -s archivo enlace.
    - Verificar: Usa ls -lh para ver el enlace simbólico (muestra una flecha hacia el destino).
    Ejemplo:
    $ ls -lh
lrwxrwxrwx 1 user user   10 Apr 27 12:00 enlace_simbolico.txt -> archivo.txt
-rw-r--r-- 1 user user 1024 Apr 27 12:00 archivo.txt
    El enlace simbólico apunta a archivo.txt.
