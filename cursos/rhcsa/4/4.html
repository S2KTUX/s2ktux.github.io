Arrancar, reiniciar y apagar un sistema normalmente

En sistemas Linux modernos, el control del estado del sistema se realiza mediante systemctl. Estos comandos aseguran un apagado limpio y evitan la corrupción de datos.

  • systemctl reboot → Reinicia el sistema
  • systemctl poweroff → Apaga el sistema
  • shutdown -h now → Apagado inmediato
  • shutdown -r +5 → Reinicia en 5 minutos

Arrancar sistemas en diferentes objetivos manualmente

Systemd utiliza targets en lugar de runlevels. Cada target define un estado operativo del sistema.

  • multi-user.target → Modo texto multiusuario
  • graphical.target → Modo gráfico
  • rescue.target → Modo rescate
  • emergency.target → Modo mínimo

Comandos clave:

  • systemctl isolate rescue.target
  • systemctl get-default
  • systemctl set-default multi-user.target

Interrumpir el proceso de arranque para obtener acceso

Durante el arranque, es posible interrumpir GRUB para acceder a un modo de recuperación.

Pasos habituales:

  1. Editar la entrada de GRUB
  2. Añadir systemd.unit=rescue.target
  3. Arrancar con Ctrl + X

Esto permite recuperar sistemas con errores de arranque o contraseñas.

Identificar procesos intensivos en CPU/memoria y eliminarlos

Linux ofrece múltiples herramientas para monitorizar procesos en tiempo real.

  • top / htop
  • ps aux
  • free -h

Para finalizar procesos:

  • kill PID
  • kill -9 PID
  • pkill nombre

Ajustar la programación de procesos

La prioridad de ejecución se controla con nice y renice.

  • nice -n 10 comando
  • renice -5 PID

Los valores van de -20 (más prioridad) a 19 (menos prioridad).

Administrar perfiles de sintonización

El servicio tuned permite optimizar el sistema según el uso.

  • tuned-adm list
  • tuned-adm active
  • tuned-adm profile throughput-performance

Localizar e interpretar archivos de registro y journals

Los logs tradicionales se encuentran en /var/log. Systemd utiliza journalctl.

  • journalctl
  • journalctl -u sshd
  • journalctl -b

Preservar journals del sistema

Por defecto, los journals pueden ser volátiles. Para hacerlos persistentes:

mkdir /var/log/journal

Esto asegura la conservación de logs tras reinicios.

Iniciar, detener y comprobar el estado de servicios de red

Los servicios se gestionan mediante systemctl.

  • systemctl start sshd
  • systemctl stop sshd
  • systemctl restart sshd
  • systemctl status sshd
  • systemctl enable sshd

Transferir archivos de forma segura entre sistemas

La transferencia segura se realiza mediante SSH.

  • scp archivo usuario@host:/ruta
  • scp usuario@host:/ruta/archivo .
  • rsync -av archivo usuario@host:/ruta
