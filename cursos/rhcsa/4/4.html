Arrancar, reiniciar y apagar un sistema

En sistemas modernos basados en systemd, la gestión del estado del sistema se realiza principalmente con systemctl. Estos comandos permiten apagar o reiniciar el sistema de forma segura.

  • systemctl poweroff → Apaga el sistema
  • systemctl reboot → Reinicia el sistema
  • shutdown -h now → Apagado inmediato
  • shutdown -r +5 → Reinicia en 5 minutos

En el examen RHCSA es importante usar métodos correctos de apagado para evitar corrupción del sistema de archivos.

Arrancar en diferentes objetivos y recuperación

Systemd reemplaza los antiguos runlevels por targets. Cada target representa un estado del sistema.

  • multi-user.target → Sistema en modo texto
  • graphical.target → Sistema con entorno gráfico
  • rescue.target → Modo rescate
  • emergency.target → Modo mínimo de recuperación

Consultar y cambiar el objetivo por defecto:

  • systemctl get-default
  • systemctl set-default multi-user.target

Durante el arranque, podemos interrumpir GRUB y añadir systemd.unit=rescue.target para obtener acceso al sistema, algo muy habitual en tareas de recuperación.

Identificar, priorizar y eliminar procesos

Linux es un sistema multitarea. Cada programa en ejecución es un proceso con su propio PID.

Herramientas principales:

  • ps aux → Lista procesos
  • top / htop → Monitorización en tiempo real
  • free -h → Uso de memoria

Finalizar procesos:

  • kill PID → Señal SIGTERM
  • kill -9 PID → Fuerza el cierre
  • pkill nombre → Mata por nombre

La prioridad de procesos se gestiona con nice y renice:

  • nice -n 10 comando → Menor prioridad
  • renice -5 PID → Cambiar prioridad

Valores de nice van de -20 (más prioridad) a 19 (menos prioridad).

Servicios, logs y transferencia de archivos

Los servicios de red se controlan con systemctl:

  • systemctl start servicio
  • systemctl stop servicio
  • systemctl restart servicio
  • systemctl status servicio
  • systemctl enable servicio

Los logs del sistema se gestionan con journalctl:

  • journalctl → Todos los logs
  • journalctl -u sshd → Logs de un servicio
  • journalctl -b → Logs del último arranque

Para preservar los logs tras reinicios:

mkdir /var/log/journal

La transferencia segura de archivos se realiza mediante SSH:

  • scp archivo usuario@host:/ruta
  • scp usuario@host:/ruta/archivo .
  • rsync -av archivo usuario@host:/ruta

Estas herramientas son fundamentales para la administración remota y aparecen frecuentemente en el examen RHCSA.
