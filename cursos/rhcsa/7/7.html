Programar tareas con cron y temporizadores systemd

En Linux podemos automatizar tareas para que se ejecuten a una hora o intervalo concreto.

La forma clásica es cron, usando crontab -e. Cada línea indica cuándo y qué comando se ejecuta.

Systemd también permite programar tareas mediante timers, que son más modernos y fáciles de controlar.

Iniciar, detener y habilitar servicios

Los servicios en sistemas modernos se gestionan con systemctl.

  • systemctl start servicio → Inicia el servicio
  • systemctl stop servicio → Detiene el servicio
  • systemctl restart servicio → Reinicia el servicio
  • systemctl status servicio → Muestra su estado
  • systemctl enable servicio → Arranca automáticamente
  • systemctl disable servicio → No arranca al iniciar

Configurar el sistema para iniciar en un objetivo específico

Systemd utiliza targets para definir cómo arranca el sistema.

  • graphical.target → Modo gráfico
  • multi-user.target → Modo texto con red
  • rescue.target → Modo rescate
  • emergency.target → Modo mínimo

Configurar clientes de servicio de tiempo

La sincronización horaria es clave en servidores. En Red Hat se usa chrony.

El servicio principal es chronyd y su configuración está en /etc/chrony.conf.

  • systemctl enable --now chronyd
  • chronyc sources
  • chronyc tracking

Instalar y actualizar paquetes de software

Red Hat utiliza dnf como gestor de paquetes.

  • dnf install paquete → Instalar
  • dnf remove paquete → Eliminar
  • dnf update → Actualizar sistema
  • dnf search paquete → Buscar

Los paquetes pueden instalarse desde repositorios remotos, CDN de Red Hat o desde archivos locales .rpm.
