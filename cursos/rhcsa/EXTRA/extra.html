1. Podman

Podman es el motor de contenedores por defecto en RHEL. A diferencia de Docker, no requiere un demonio (daemonless) y permite ejecución segura.

  • Rootless: Ejecuta contenedores como usuario normal para mejorar la seguridad.
  • Persistencia: Es vital saber generar archivos de unidad para systemd con podman generate systemd para que los contenedores arranquen tras un reboot.
  • Imágenes: Familiarízate con el registro local y la gestión de volúmenes para persistencia de datos.

2. Find

El comando find es esencial para localizar archivos bajo criterios de búsqueda complejos que suelen pedirse en los enunciados.

  • find / -user [nombre]: Buscar archivos por propietario.
  • find / -size +10M -size -50M: Filtrar por un rango de tamaño específico.
  • find / -perm -4000: Localizar archivos con el bit SUID activo.
  • find / -mmin -120: Buscar archivos modificados en los últimos 120 minutos.

3. Reset contraseña root

Si pierdes el acceso o necesitas resetear la clave al inicio, el procedimiento de rd.break es el estándar en RHEL:

  1. Reinicia y edita el arranque en GRUB (tecla e).
  2. Añade rd.break al final de la línea linux.
  3. Monta el sistema: mount -o remount,rw /sysroot.
  4. Accede al entorno: chroot /sysroot.
  5. Cambia la clave: passwd root.
  6. Etiquetado SELinux: touch /.autorelabel.
  7. Finaliza con exit dos veces.

4. Video Inside REDHAT

Es fundamental visualizar el video oficial "Inside a Red Hat Certification" para conocer el entorno real del examen.

  • El entorno es una interfaz basada en web que simula un escritorio con terminales.
  • Las instrucciones no siempre detallan cómo abrir la consola de cada máquina; debes familiarizarte con el selector de VMs del panel lateral.
  • Practica la navegación entre las diferentes máquinas virtuales asignadas (node1, node2, etc.).

5. Crear user y meterle en wheel para hacer ssh y poder copy paste mejor

Acceder directamente como root por SSH suele fallar. La mejor técnica para ganar velocidad es:

  • Crea un usuario de gestión: useradd admin-exam.
  • Añádelo al grupo wheel: usermod -aG wheel admin-exam.
  • Opcional: Configura NOPASSWD:ALL en visudo para este usuario.
  • Accede desde la máquina principal por SSH. Esto te permitirá **copiar y pegar** directamente desde el navegador de instrucciones (URLs, nombres largos de archivos, etc.) hacia tu terminal, evitando errores de escritura.
