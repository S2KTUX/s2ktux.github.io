Configurar direcciones IPv4 e IPv6

En RHEL, la gestión de red se realiza principalmente a través de NetworkManager utilizando la herramienta de línea de comandos nmcli. Tambien podemos usar el comando nmtui para modificar con interfaz grafica o hacer cambios temporales con el comando ip.

Con nmcli, que es el que principalmente usaremos, podemos crear una red, modificar, eliminar y gestionar el estado de las interfaces de forma dinámica.s el que principalmente usaremos, podemos crear una red, modificar, eliminar

Una vez aplicados los comandos, usamos nmcli con up "ID" para activar la nueva configuración. Podemos verificar los cambios con ip addr show.

Configurar la resolución del nombre de host

El nombre de host (hostname) identifica al equipo en la red. Para que el cambio sea permanente en RHEL, utilizamos hostnamectl.

Además, para la resolución local sin DNS, editamos el archivo /etc/hosts.

  • Cambiar hostname: hostnamectl set-hostname nodo1.ejemplo.com
  • Verificar: hostnamectl status

Para añadir servidores DNS a una conexión específica: nmcli con mod "ID" ipv4.dns "8.8.8.8 1.1.1.1". Esto actualizará automáticamente el archivo /etc/resolv.conf al levantar la conexión.

Servicios de red al inicio del sistema

Para asegurar que nuestra configuración de red sea efectiva desde el arranque, el servicio NetworkManager debe estar habilitado.

Utilizamos los comandos de systemctl que ya conocemos para gestionar el demonio de red:

  • Habilitar inicio automático: systemctl enable NetworkManager
  • Verificar estado: systemctl is-enabled NetworkManager

Si creamos servicios personalizados que dependen de la red, debemos usar la directiva After=network-online.target en la sección [Unit] del archivo del servicio.

Restringir acceso con firewalld y firewall-cmd

La seguridad en RHEL se gestiona mediante firewalld, que organiza las reglas en "zonas". La zona por defecto suele ser public.

Para que los cambios sobrevivan a un reinicio, siempre debemos usar el flag --permanent y posteriormente recargar el servicio.

  • Listar reglas actuales: firewall-cmd --list-all
  • Permitir servicio: firewall-cmd --permanent --add-service=http
  • Permitir puerto: firewall-cmd --permanent --add-port=8080/tcp
  • Recargar configuración: firewall-cmd --reload

Si nos piden restringir el acceso, podemos eliminar servicios con --remove-service o usar reglas enriquecidas (rich rules) para filtrar por IPs específicas.
