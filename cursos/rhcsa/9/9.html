Crear, eliminar y modificar cuentas de usuario locales

En RHEL, la administración de usuarios es crítica para la seguridad. Los archivos principales involucrados son /etc/passwd (datos del usuario) y /etc/shadow (contraseñas cifradas).

Utilizamos comandos estándar para gestionar el ciclo de vida de las cuentas:

  • Crear: useradd -c "Nombre Real" -s /bin/bash usuario. El flag -m crea el home automáticamente.
  • Modificar: usermod permite cambiar el nombre, el shell o bloquear la cuenta (-L).
  • Eliminar: userdel -r usuario. El flag -r es fundamental para borrar también su directorio personal y buzón de correo.

Ajustar el envejecimiento (aging) de la contraseña

Para cumplir con normativas de seguridad, debemos forzar a los usuarios a cambiar sus credenciales periódicamente. La herramienta principal para esto es chage.

Podemos configurar parámetros como la expiración, el aviso previo y la inactividad:

Comandos comunes:

  • chage -l usuario: Lista los detalles actuales de expiración.
  • chage -M 90 usuario: Establece un máximo de 90 días para cambiar la clave.
  • chage -d 0 usuario: Obliga al usuario a cambiar la contraseña en su próximo inicio de sesión.

Crear, eliminar y modificar grupos locales

Los grupos permiten asignar permisos a varios usuarios simultáneamente. Un usuario puede pertenecer a un grupo primario (GID en /etc/passwd) y a múltiples grupos secundarios.

Operaciones principales:

  • Crear grupo: groupadd nombre_grupo.
  • Añadir usuario a grupo: usermod -aG grupo usuario. ¡OJO! El -a (append) es vital; si lo olvidas, sacarás al usuario de todos sus otros grupos.
  • Eliminar grupo: groupdel nombre_grupo.

Configurar el acceso privilegiado

En lugar de compartir la contraseña de root, utilizamos sudo para delegar privilegios de forma controlada. La configuración reside en /etc/sudoers y en el directorio /etc/sudoers.d/.

Para editar estos archivos de forma segura, usamos visudo, que verifica la sintaxis antes de guardar para evitar que perdamos el acceso al sistema.

  • El grupo wheel: Por defecto en RHEL, cualquier usuario en el grupo wheel tiene permisos de administrador.
  • Configuración personalizada: usuario ALL=(ALL) NOPASSWD: ALL permite ejecutar cualquier comando sin pedir contraseña (común en automatizaciones).

Como buena práctica, siempre es mejor crear un archivo pequeño dentro de /etc/sudoers.d/nombre_usuario que editar el archivo principal.
