Configurar el acceso a los repositorios de RPM

Los repositorios RPM son fuentes de software que permiten instalar y actualizar paquetes mediante gestores como dnf. En sistemas RHEL, Rocky o AlmaLinux, los repositorios se definen mediante archivos .repo ubicados en /etc/yum.repos.d/.

Comandos básicos para trabajar con repositorios:

  • dnf repolist → Lista los repositorios habilitados
  • dnf config-manager → Habilitar o deshabilitar repositorios
  • subscription-manager → Gestión de repositorios en RHEL

Instalar y eliminar paquetes de software RPM

El gestor de paquetes principal en sistemas modernos basados en RPM es dnf. Permite resolver dependencias automáticamente y mantener el sistema actualizado.

  • dnf install paquete → Instalar un paquete
  • dnf remove paquete → Eliminar un paquete
  • dnf update → Actualizar el sistema
  • dnf search nombre → Buscar paquetes
  • rpm -qa → Listar paquetes instalados

Configurar el acceso a los repositorios de Flatpak

Flatpak es un sistema de distribución de software independiente de la distribución, diseñado para aplicaciones de escritorio con dependencias aisladas.

El repositorio más común es Flathub, que puede añadirse mediante:

  • flatpak remote-add → Añadir repositorios
  • flatpak remotes → Listar repositorios configurados

Instalar y eliminar paquetes de software Flatpak

Una vez configurado un repositorio Flatpak, podemos gestionar aplicaciones de forma sencilla y segura.

  • flatpak install app → Instalar una aplicación
  • flatpak uninstall app → Eliminar una aplicación
  • flatpak list → Listar aplicaciones instaladas
  • flatpak update → Actualizar aplicaciones Flatpak
