Acceder a un prompt de shell y ejecutar comandos con la sintaxis correcta

Un prompt de shell, tecnicamente hablando, es una interfaz de texto dinámica que actúa como un resumen del estado actual del entorno de trabajo. El prompt indica tu identidad y ubicación mediante el formato usuario@máquina:ruta, cerrando con $ para usuarios normales o # para privilegios de root.

El acceso a una shell puede realizarse desde consola local usando Ctrl + Alt + F1–F6, de forma remota mediante SSH o abriendo la app Terminal si estamos en interfaz gráfica, etc....

Cuando nos piden ejecutar con la sintaxis correcta se refieren a escribir correctamente los comandos, teniendo muy en cuenta que Linux es "Case Sensitive" (distingue entre mayusculas y minusculas). La sintaxis estándar es:

comando [opciones] [argumentos]

Usar la redirección de entrada y salida (>, >>, |, 2>, etc.)

En Linux todo es un archivo y cada proceso utiliza descriptores estándar:

  • 0 -> STDIN (Entrada - Lo que escribimos por pantalla)
  • 1 -> STDOUT (Salida - Lo que nos devuelve un comando)
  • 2 -> STDERR (Errores - Nos imprime los posibles errores de un comando, en caso de que los haya)

Podemos utilizar la redirección para controlar hacia dónde fluye la información :

  • < , << , <<< -> Entrada desde archivo, bloque de texto o cadena.
  • > , >> -> Salida a archivo (sobrescribir o añadir).
  • 2> -> Captura exclusiva de errores.
  • &> -> Redirección completa (salida + errores).
  • | -> Tubería para conectar la salida de un comando con la entrada de otro.

Usar grep y expresiones regulares para analizar texto

grep sirve para analizar y buscar patrones dentro de archivos o salidas de comandos. Podemos utilizar grep tanto directamente o a traves de la redirección con pipe (|). Las opciones (flags) más importantes son :

  • -E -> Permite usar expresiones regulares extendidas. Ejemplo: grep -E "error|fallo" log.txt
  • -v -> Muestra todas las líneas que no coinciden con el patrón (inversión). Ejemplo: grep -v "completado" tareas.txt
  • -i -> Ignora la diferencia entre mayúsculas y minúsculas. Ejemplo: grep -i "linux" manual.txt

Expresiones regulares básicas cuando usamos grep -E :

  • ^ -> Indica que la línea debe empezar por ese texto.
  • $ -> Indica que la línea debe terminar por ese texto.
  • . -> Representa cualquier carácter.
  • * -> Lo anterior puede aparecer muchas veces o ninguna.
  • [] -> Busca cualquiera de los caracteres que haya dentro.
  • | -> Busca una opción u otra ( grep -E "error|fallo")
  • \ -> Sirve para buscar símbolos especiales como si fueran texto normal( grep \.archivo nos buscará .archivo)

Acceder a sistemas remotos mediante SSH

SSH es un protocolo de red diseñado para establecer conexiones seguras entre dos maquinas. El puerto por defecto es el 22 y su archivo de configuración es /etc/ssh/sshd_config ( a nivel global ) o ~/.ssh/config ( a nivel local) . Las flags más importante serían :

  • -p -> Permite especificar un puerto.
  • -l -> Especifica el nombre de usuario con el que se desea iniciar sesión en la máquina remota.
  • -v -> Verbose, nos muestra más información sobre el proceso de conexión.
  • -i -> Indica la ruta de la clave privada (Identity file).

Iniciar sesión y cambiar de usuario en destinos multiusuario

Root es la cuenta de superusuario o administrador del sistema, teniendo privilegios totales. Para cambiar de usuario se utilizan el comando su. Podemos usarlo con guión o sin guión, aunque generalmente lo querremos usar siempre con guión para cargar el entorno completo (variables de entorno, PATH, etc.)

Podemos utilizar además el comando sudo -i, que a diferencia de su, usaremos la contraseña del usuario actual (Deberemos tener permisos sudo). Por otro lado, podemos utilizar sudo -u user COMANDO para lanzar un comando desde root.

Por ultimo, para comprobar que usuario somos podemos utlizar id o whoami.

Archivar, comprimir, desempaquetar y descomprimir archivos utilizando tar, gzip y bzip2

tar se utiliza para empaquetar archivos (sin reducir espacio), mientras que gzip y bzip2 comprimen (reduciendo el espacio). Las flags de tar son :

  • -c -> Create: Crea un nuevo archivo de empaquetado.
  • -x -> Extract: Extrae el contenido de un archivo empaquetado existente.
  • -v -> Verbose: Muestra el listado de archivos procesados por pantalla.
  • -f -> File: Indica el nombre del archivo resultante o el archivo a extraer (siempre debe ser la última flag).

Comprimir o descomprimir con gzip o bzip directamente desde tar utilizando las flags :

  • -z -> Gzip: Comprime o descomprime el archivo usando gzip (.tar.gz).
  • -j -> Bzip2: Comprime o descomprime el archivo usando bzip2 (.tar.bz2).

Por otro lado, también podemos comprimir directamente con gzip o bzip2 o descomprimir con gunzip o bunzip2:

  • gzip / gunzip Comprimir / Descomprimir en el algoritmo gzip. Ejemplo: gzip file.txt / gunzip file.txt.gz
  • bzip2 / bunzip2 Comprimir / Descomprimir en el algoritmo bzip2. Ejemplo:bzip2 file.txt / bunzip2 file.txt.bz2

Crear y editar archivos de texto

Los archivos pueden crearse y editarse con distintas herramientas. Las más utilizadas son nano y vi, siendo esta ultima la que mayor curva de aprendizaje tiene.

Lo que debemos de tener en cuenta con VI principalmente son sus modos y comandos básicos:

  • Modo inserción -> Permite escribir texto. Entramos presionando i (inserta antes del cursor) o a (inserta después del cursor). Con Esc salimos al modo comando
  • Borrar una línea -> En modo comando, pulsar dd para eliminar la línea actual.
  • Buscar -> En modo comando, pulsar / seguido del texto a buscar.
  • Guardar y salir -> En modo comando, escribiremos :wq!.
  • Salir sin guardar -> En modo comando, escribir :q! para descartar todos los cambios.
  • Deshacer -> En modo comando, pulsar u para deshacer la última acción.

Crear, eliminar, copiar y mover archivos y directorios

Linux dispone de comandos básicos para la gestión de archivos:

  • mkdir -> Crea uno o varios directorios. Ejemplo: mkdir -p /tmp/ruta/nueva.
  • rmdir -> Elimina directorios, pero solo si están vacíos. Ejemplo: rmdir carpeta_vacia
  • touch -> Crea un archivo vacío o actualiza la fecha de acceso/modificación de uno existente. Ejemplo:touch archivo.txt
  • cp -> Copia archivos o directorios. Ejemplo: cp -r /home/user/documentos /tmp/backup (la flag -r es obligatoria para directorios).
  • mv -> Mueve o renombra archivos y directorios. Ejemplo: mv viejo_nombre.txt nuevo_nombre.txt
  • rm -> Elimina archivos o directorios. Ejemplo: rm -rf /tmp/directorio_inservible (la flag -r borra directorios y -f evita confirmaciones).

Crear enlaces físicos y simbólicos

Un enlace físico es simplemente un nombre adicional para los mismos datos en el disco (mismo inodo), mientras que un enlace simbólico es un acceso directo que apunta a la ruta del archivo original (acceso directo en Windows). La sintaxis es ln (-s) /ruta/al/original /ruta/al/enlace

  • ln -> Crea un enlace físico (hard link) al archivo original. Ejemplo: ln archivo.txt link_fisico
  • ln -s -> Crea un enlace simbólico (soft link) o acceso directo al archivo original. Ejemplo: ln -s archivo.txt link_simbolico

Las diferencias son que el enlace duro se mantiene la información en el enlace aunque borremos el archivo original y en el simbolico se pierde la información cuando borramos el original ( como vemos en la imagen, se rompe el enlace)

Enumerar, configurar y cambiar permisos estándar ugo/rwx

Los permisos controlan el acceso a archivos y directorios. Estos están divididos en 3 "bloques" : PROPIETARIO - GRUPO - OTROS. Cada bloque utiliza tres permisos básicos que se pueden representar con letras (simbólico) o números (octal):

  • Lectura (r) -> Permite leer el contenido del archivo o listar el contenido del directorio. Valor numérico: 4.
  • Escritura (w) -> Permite modificar o eliminar el archivo, o crear/eliminar archivos dentro de un directorio. Valor numérico: 2.
  • Ejecución (x) -> Permite ejecutar un archivo (si es un script o programa) o entrar en un directorio. Valor numérico: 1.

Para comprobar los permisos de un directorio o un archivo, utilizaremos ls -l donde podremos listar los diferentes archivos y/o carpetas que existan en esa ruta. Una vez definidos, los podemos modificar usando los siguientes comandos :

  • chmod -> Cambia los permisos de un archivo o directorio (el "modo").Ejemplo numérico:chmod 750 archivo.txt (rwx para propietario, r-x para grupo, --- para otros). Ejemplo simbólico: chmod u+x,g-w,o=r archivo.txt
  • chown -> Cambia el propietario (usuario y/o grupo) de un archivo o directorio. Ejemplo solo usuario: chown juan archivo.txt. Ejemplo usuario y grupo: chown juan:sales archivo.txt
  • -R -> Flag común para ambos comandos que aplica los cambios de forma recursiva a todos los subdirectorios y archivos dentro de una ruta. Ejemplo: chmod -R 644 /var/www/html

Localizar, leer y utilizar la documentación del sistema, incluyendo man, info y archivos en /usr/share/doc

Linux dispone de documentación integrada para casi todos los comandos. Tenemos diferentes opciones :

  • man (manual pages) -> Proporciona la documentación estándar y concisa de un comando. Es la herramienta principal.
  • info -> Ofrece documentación más detallada e hiperenlazada que man.
  • /usr/share/doc -> Directorio que contiene documentación extendida, ejemplos, READMEs ...
  • --help -> Un parámetro rápido que casi todos los comandos soportan para ver rápidamente sus opciones básicas.
  • whatis -> Pequeña explicación de lo que hace el comando
