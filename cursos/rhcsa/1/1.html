Acceder a una solicitud de shell y sintaxis correcta

Podemos acceder a una consola local usando Ctrl + Alt + F1 hasta F6 e iniciar sesión con usuario y contraseña.

También podemos acceder mediante SSH:

ssh usuario@192.168.23.5

La sintaxis correcta de los comandos es:

comando + opciones + argumentos
ls -l /ruta
  • Usar comillas cuando hay espacios: mkdir "Mi carpeta"
  • Variables en mayúsculas: VAR=hola; echo $VAR
  • Linux distingue mayúsculas y minúsculas

Redirección de entrada y salida

Cada proceso tiene tres descriptores:

  • 0 → STDIN
  • 1 → STDOUT
  • 2 → STDERR
cat /etc/passwd > archivo.txt
cat /etc/passwd >> archivo.txt
ls /ruta-inexistente 2> error.txt
ls /noexiste /etc > salida.txt 2>&1
comando &> salida.txt

Redirección de entrada:

wc -l < archivo.txt

Here Document:

cat << EOF > archivo.txt
contenido
EOF

Uso de pipe:

ls -l /tmp | grep ".conf"

Uso de grep y expresiones regulares

grep root /etc/passwd
  • -i → ignora mayúsculas
  • -v → excluye coincidencias
  • -n → número de línea
  • -c → cuenta coincidencias
  • -r → búsqueda recursiva
  • -o → solo coincidencia

Expresiones regulares básicas:

  • ^root → comienza por
  • bash$ → termina en
  • r..t → cualquier carácter
  • roo*t → cero o más repeticiones
  • r[ao]t → rango
  • [0-9] → números

Archivar y comprimir archivos

Compresión individual:

bzip2 -k archivo.txt
