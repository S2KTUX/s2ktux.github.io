108.1 Mantener la hora del sistema

El comando date muestra la hora y fecha actual del sistema y permite ajustarla manualmente. timedatectl ofrece una alternativa para verificar y configurar la hora.

Reloj de hardware

El comando hwclock interactúa con el reloj en tiempo real (RTC) del hardware, manteniendo la hora incluso si el sistema está apagado.

Ejemplos timedatectl:

  • Listar zonas: timedatectl list-timezones
  • Ajustar hora: timedatectl set-time "2025-05-24 12:30:00"

Zona horaria

La zona horaria define el desfase respecto a UTC. Los datos están en /usr/share/zoneinfo/ (ej. Canada/Eastern). /etc/localtime suele ser un enlace simbólico a uno de estos archivos.

Configuración manual: sudo ln -sf /usr/share/zoneinfo/Canada/Eastern /etc/localtime.

Una zona correcta es clave para convertir marcas de tiempo Unix a hora local.

Demonio NTP

El protocolo NTP sincroniza la hora con servidores de red. ntpd se ejecuta en segundo plano. Su configuración es /etc/ntp.conf.

ntpd sondea múltiples servidores y selecciona los más precisos para ajustar la hora.

Chrony

Alternativa moderna a ntpd, con mejor precisión y soporte para sincronización intermitente. Configuración en /etc/chrony.conf.

Fuentes remotas: pool y server.

pool pool.ntp.org iburst

La opción iburst acelera la sincronización inicial.

108.2 Registros del sistema

Los registros almacenan eventos en orden cronológico. rsyslog y systemd-journald son los principales servicios. Facilidades: mail, local5, etc.

Gestión con rsyslog

rsyslog usa modelo cliente-servidor. Configuración en /etc/rsyslog.conf. Registros en /var/log/.

  • /etc/rsyslog.conf: Define reglas (<facilidad>.<prioridad> <acción>).
  • Directorios /var/log/:
    • syslog: Mensajes generales.
    • auth.log: Autenticación.
    • kern.log: Kernel.
    • messages: Diversos (destino por defecto).
  • logger: Añade mensajes manualmente (ej. logger "Mensaje de prueba").

Rotación con logrotate

Evita que los registros ocupen demasiado espacio. logrotate se ejecuta vía /etc/cron.daily/logrotate.

  • Configuración: /etc/logrotate.conf (global) y /etc/logrotate.d/ (específica).
  • Opciones comunes: `weekly`, `rotate 4`, `compress`.

Gestión con systemd-journald

Centraliza registros en formato binario. Almacenamiento en /run/log/journal/ (volátil) o /var/log/journal/ (persistente).

  • journalctl: Consulta el journal (filtros: --since, -u, -b).
  • systemd-cat: Envía mensajes al journal (ej. echo "Mensaje" | systemd-cat).
  • /etc/systemd/journald.conf: Configura ubicación (Storage=) y límites.

108.3 Conceptos básicos del Agente de Transferencia de Correo

Un MTA envía y recibe correos en Linux, guardándolos en bandejas privadas (ej. `/var/spool/mail/usuario`). Usa SMTP (puerto TCP 25).

MTAs en Linux: sendmail, postfix, exim

  • sendmail: Tradicional, flexible pero complejo. Por defecto solo local. Para aceptar remotas, editar /etc/mail/sendmail.mc en `DAEMON_OPTIONS` y reiniciar servicio.
  • postfix: Moderno, sencillo de configurar. Cola en /var/spool/postfix/.
  • exim: Popular en Debian, flexible.

Los MTAs modernos son compatibles con comandos de sendmail.

Herramientas y comandos del MTA

  • sendmail emulation: sendmail -bp (lista cola), sendmail -q (reintenta).
  • mailq: Muestra cola pendiente y motivo de retraso (ej. "Connection refused").
  • mail: Cliente consola. Modos lectura (`mail`) y envío (`mail -s "Asunto" user <<< "Msg"`).
  • newaliases: Actualiza base de datos de alias tras modificar /etc/aliases.

Personalización de la entrega

  • ~/.forward: Archivo en el directorio personal para reenviar correos. No necesita `newaliases` y debe ser escribible solo por el propietario.
  • /etc/aliases: Alias del sistema (ej. `postmaster: carol`). Requiere `newaliases`.

108.4 Gestión de la impresión y las impresoras

A pesar del "papelless office", la impresión es clave. CUPS facilita esto enviando archivos a una cola, procesándolos con filtros.

Servicio CUPS y su configuración

Instalación: sudo apt install cups (Debian) o sudo dnf install cups (Fedora), luego sudo systemctl start cups.service.

Configuración en /etc/cups/:

  • cupsd.conf: Configura servicio, acceso web (WebInterface Yes).
  • printers.conf: Configuración de impresoras (no editar estando activo).
  • printcap: Archivo legacy LPD, enlace a /run/cups/printcap.
  • ppd/: Archivos de descripción de impresora.

Logs en /var/log/cups/. Interfaz web en http://localhost:631.

Herramientas legacy de lpd

  • lpr: Envía a impresora por defecto.
  • lprm: Cancela trabajos (ej. lprm 20).
  • lpq: Muestra cola y estados (ej. lpq -a todas las colas).
  • lpstat: Estado de impresoras (ej. lpstat -p disponibles).
